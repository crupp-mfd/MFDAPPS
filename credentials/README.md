# Credentials Directory

Leg hier alle sensiblen Zugangsdaten für lokale Entwicklung ab.

- `ionapi/` für Infor-Service-Accounts (`*.ionapi` Dateien)
- Weitere Unterordner kannst du nach Bedarf ergänzen (z.B. `env/`, `certs/`).
- Diese Dateien sollten **nicht** ins Git-Repo committet werden.

> Tipp: Hinterlege hier eine Beispiel-Datei (z.B. `service_account.ionapi.example`) mit Platzhalterwerten, damit andere wissen, welche Keys benötigt werden.
